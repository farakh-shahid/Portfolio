## Problem

Burst-heavy telemetry can overwhelm synchronous processing pipelines.

## Approach

- Split ingestion from processing with queue boundaries.
- Added idempotency keys and dead-letter handling.
- Introduced traceability for per-message processing outcomes.

## Outcome

Improved resilience and observability while maintaining consistent processing SLAs.
